# Code Audit Report: microstructure-tpw (branch `feat/perturb-plot-profiles`, PR#73)

## 1. Executive Summary

This audit synthesizes **54 confirmed findings** across the RSI I/O, chi/epsilon physics, perturb pipeline, plotting, and processing subsystems: **0 Blocker, 23 Major, 25 Minor, 11 Nit** (5 of the items are dual-tracked as the test-gaps that hide a paired code defect). Several findings overlap (notably the `np.trapezoid`/numpy-floor packaging defect, the dateline-unsafe longitude handling, and the `np.mean`-vs-`np.nanmean` viscosity-poisoning family), so the de-duplicated set is somewhat smaller. **Overall risk read: moderate-to-elevated, but heavily conditional.** The default ARCTERX/SN479 production path (stable input set, fresh output dir, mean aggregation, near-145┬░E geography, binary-exact `bin_size`, numpy 2.x) is largely safe; the major risks cluster in *opt-in or off-nominal* configurations тАФ `files.merge=true` (fuses 29 independent casts), median binning (LnSigma quadrature understated), non-epoch/NaN-bearing hotel/GPS inputs (silent fill-value and time-decode corruption), the systematic drop-the-max probe-rejection low bias, and the chi viewer/plot paths (field-order swap, depth-grid misregistration). The single most portable hazard is the declared `numpy>=1.26` floor against pervasive `np.trapezoid` (NumPy 2.0+ API), which would crash the entire chi/epsilon subsystem on a conformant 1.26 install. Most defects are silent (wrong numbers, dropped data, mislabeled CF metadata) rather than loud, which raises their practical severity for a scientific product where outputs are trusted without an error to flag them.

## 2. Findings by Severity

### Major

- **`pyproject.toml:17` (numpy floor) тАФ packaging/correctness.** Declared `numpy>=1.26,<3` permits 1.26.x, but the code calls `np.trapezoid` (NumPy 2.0+ rename of `np.trapz`) at ~17 sites across `chi/chi.py` (all three chi paths + `_variance_correction`), `chi/l4_chi.py:246`, `rsi/quick_look.py:145`, and `scor160/l4.py` (366/411/412/522). On any 1.26 install every chi method, the chi L4 stage, quick-look, and shear-L4 variance integrals raise `AttributeError`. Latent only because the dev env is numpy 2.4.x. **Fix:** raise the floor to `numpy>=2.0,<3` (and/or add `trapezoid = getattr(np, "trapezoid", np.trapz)` shim); add a lowest-numpy CI matrix entry. *(Merges the narrower `scor160/l4.py:366`-only report.)*

- **`perturb/merge.py:99-114` тАФ correctness.** `find_mergeable_files` chains files on `(config_hash, endian, record_size)` + sequential `file_number` only, with no time-continuity or `restarted`-flag check. Verified on real data: all 29 ARCTERX SN479 casts share one config hash and carry sequential file numbers, so `find_mergeable_files` returns ONE 29-file chain. With `files.merge=true` the entire deployment of independent casts is byte-concatenated into one `.p`, destroying per-cast structure. **Fix:** require start_time continuity (within tolerance) and/or `restarted==0` on the continuation file before chaining.

- **`perturb/merge.py:55-57` тАФ correctness.** `_file_group_key` omits `header_size`/`config_size`. Two files sharing config_hash+record_size but differing in `header_size` still chain; the splice skips each file's own header geometry while the reader reparses the merged file with the FIRST file's geometry, silently mis-slicing all appended records with no partial-record warning (verified: merged size stays a clean multiple). **Fix:** include `header_size`/`config_size` in the key, or assert per-record geometry identical to `chain[0]` before splicing.

- **`perturb/trim.py:116-121` тАФ correctness.** Trim skip-guard reuses a stale truncated output whenever `dest.mtime >= source.mtime` with no size/hash/geometry check. A repaired/completed source restored with an equal/older mtime (`cp -p`, rsync, tar extract, backup restore, NFS skew) is silently skipped; downstream merge reads header geometry and config from the truncated dest, never revalidating the repaired source. **Fix:** make the skip content-aware тАФ re-read the source header, recompute expected trimmed size, and only skip when size + geometry match.

- **`processing/epsilon_combine.py:117-136` тАФ oceanographic.** Iterative 95% CI removal unconditionally NaNs the **maximum** probe each outside row (`nanargmax`), never a low outlier. For 2-probe VMP, whenever probes disagree beyond the CI the lower probe always wins тЖТ systematic low bias in `epsilonMean` (verified: probes [1e-7, 5e-9] тЖТ mean 5e-9 vs true geomean ~2.2e-8; a low-FOM-clean fouled probe is unmitigated upstream). Same pattern in `chi_combine.py:147-163`. **Fix:** remove the probe whose `ln(eps)` is furthest from the cross-probe ln-mean/median (symmetric), re-deriving the threshold from survivors each pass.

- **`processing/ct_align.py:108-113` тАФ correctness.** Cross-profile lag uses `argmin(|cum_w - total/2|)`, the *closest-to-half* index, not the weighted median (first index reaching тЙе half). Diverges whenever weights are skewed: lags [-2,3] w[1,10] тЖТ returns -2 (correct +3). A single long high-correlation cast that should dominate alignment is ignored, mis-aligning C/T for the whole file тЖТ corrupted salinity/density. **Fix:** true weighted median via `searchsorted(cum_w, total/2, side='left')`.

- **`processing/ct_align.py:80-94,105-118` тАФ error-handling.** A flatlined/constant T or C segment gives `norm==0`; the `if norm>0` guard skips normalization but not the profile, so all-zero `corr` тЖТ `argmax`=0 тЖТ the most-negative lag is recorded. With all-degenerate (or even one degenerate among valid, via argmin tie) the function applies the full `-max_lag_seconds` shift (verified: тИТ5 s / 320-sample shift), corrupting the whole conductivity record silently. **Fix:** `continue` when `norm==0`/`max_corr==0`; return `C.copy(), 0.0` on zero total weight; fix the tie-break.

- **`chi/chi.py:538-582` (and 579-605) тАФ correctness.** In the iterative (default) chi fit, a first-iteration kB failure (`kB_best` NaN or <1) breaks the loop leaving `epsilon=NaN`, `fom=NaN`, but returns the finite band-integrated `chi=chi_obs`. That finite chi enters `_compute_chi_final`'s geomean (checks only finite & >0) and escapes `_apply_fom_cut` (which skips NaN fom; and is off by default), with no K_max_ratio cut for chi anywhere. A wholly-failed Batchelor fit thus contributes a spurious noise-driven chi to `chiMean`/K_T/╬У. **Fix:** return `chi=NaN` when `kB` is non-finite (or <1), or gate `_compute_chi_final`/`mk_chi_mean` on finite kB/fom. *(Two reports of the same defect merged.)*

- **`chi/l3_chi.py:149` тАФ numerical.** Per-window slow temperature uses `np.mean` (not `nanmean`): one NaN sample тЖТ `T_means[w]=NaN` тЖТ `nu=NaN` (viscosity) and corrupted noise model тЖТ `chi=NaN` for affected windows, silently. NaN source is real (missing T1 defaults to all-NaN in `adapter.py:74`; `np.interp` propagates). Asymmetric with epsilon (`scor160/l4.py:154-160` substitutes 10 ┬░C + warns). **Fix:** `np.nanmean` for `T_means` and a NaNтЖТfallback temperature guard mirroring the epsilon side.

- **`chi/l3_chi.py:154` тАФ numerical.** Same class for salinity: per-window `np.mean` over salinity with a single NaN (routine at profile start when conductivity cell is in air/unflushed; `gsw.SP_from_C` returns NaN) тЖТ `nu=NaN` тЖТ corrupted chi in near-surface windows, where mixing studies most want data. Affects all three measured-salinity producers (`adapter.py`, `perturb/pipeline.py:795`, `chi_io.py:151`). **Fix:** `np.nanmean` for `sal_means`; sanitize/clamp salinity where produced (mirror the epsilon temp fallback).

- **`perturb/binning.py:304-308` (and 563-565) тАФ numerical.** `*LnSigma` (epsilon/chiLnSigma) combine is `sqrt(_bin_array(sigma**2, agg))`, which equals the documented quadrature RMS only when `agg=nanmean`. Under **median** binning (`aggregation: median`, user-reachable) it becomes `sqrt(median(sigma**2))` тАФ for odd counts a no-op collapsing to `median(|sigma|)`; verified ~4├Ч understatement of combined log-uncertainty. **Fix:** always average variances with `np.nanmean` for the LnSigma branches regardless of `agg` (or correct the label/comment if a robust statistic is intentional).

- **`perturb/pipeline.py:1881` тАФ correctness.** Binning globs the whole shared, content-hashed-and-reused output dir (`output_dirs["profiles"].glob("*.nc")`; same for diss/chi). Per-profile NCs accumulate across runs and are never cleaned, so re-running with the same config but a *reduced/changed* input set silently folds stale profiles into `binned.nc`/combos. The exact per-run paths are available in `result[...]` but discarded. **Fix:** bind binning inputs to this run's `result["profiles"]/["diss"]/["chi"]` paths, or stamp/prune by an input-set manifest.

- **`perturb/plot/eps_chi.py:217` тАФ correctness.** Combo path loads chi's own depth axis into `_depth_chi` then discards it; the realignment block only fixes the profile (column) axis, never the bin (depth) row axis. When chi vs eps bin grids differ in **length** тЖТ `chi/eps` broadcast ValueError aborts the figure; when they match in length but differ in **centers** (routine, since FP07 chi profiles are trimmed/calibration-rejected differently than shear) тЖТ chi and chi/eps panels are silently plotted ~1 bin too shallow. The legacy per-profile path correctly re-bins; the combo path dropped it. **Fix:** reindex chi onto eps's depth grid (or assert bin-for-bin equality and error clearly). *(Subsumes the narrower `eps_chi.py:297` broadcast-only report.)*

- **`perturb/gps.py:250,280-281` тАФ oceanographic.** `t_var[:].data`/`lat_var[:].data`/`lon_var[:].data` expose netCDF `_FillValue` (e.g. тИТ999) as real numbers because auto-masking is not disabled and `.data` bypasses the mask (verified). A тИТ999 lat/lon or fill-valued time feeds `interp1d` тЖТ wildly wrong interpolated positions тЖТ corrupted TEOS-10 salinity and geospatial attrs. **Fix:** `.filled(np.nan)` for masked arrays (time, lat, lon), or `set_auto_mask(False)` only after mapping fills to NaN.

- **`perturb/hotel.py:281,293,303` тАФ oceanographic.** Same `.data` fill-value exposure for hotel default time, every data channel, and per-channel time columns. Glider/AUV telemetry gaps stored as fills become e.g. тИТ999 and are interpolated onto the instrument axis as physical speed/pitch/CTD, propagating into dissipation/chi. **Fix:** `.filled(np.nan)` (or a fillтЖТNaN helper) for time, each variable, and per-channel time arrays.

- **`perturb/hotel.py:152-167` тАФ numerical.** `time_format='auto'` (default) feeds raw numeric arrays to `pd.to_datetime` in the ambiguous `[1e6,1e9)` band (treated as nanoseconds тЖТ silent garbage, no exception). Worse, `np.median` (not `nanmedian`) returns NaN if any time sample is NaN, so **every** file with a single NaN timestamp falls through to the nanosecond misinterpretation. **Fix:** use `np.nanmedian`; do not feed numeric arrays to `pd.to_datetime` in the ambiguous band; restrict ISO parse to non-numeric dtypes; raise or treat-as-relative for ambiguous numeric magnitudes.

- **`rsi/convert.py:64` тАФ correctness.** `_classify_channels` selects magnetometers with `type == "mag"`, but the ODAS/config/converter type is `"magn"`. `mag_names` is always empty, so the CF MAG variable and `N_MAG_SENSORS` dimension are never written; magnetometer data falls through as a generic supplementary variable, losing CF role/metadata. **Fix:** compare to `"magn"`.

- **`perturb/plot/layout.py:233-237` тАФ correctness.** `plot_columns` calls `strictly_increasing()` (nudges ties by ~1e-3┬╖median spacing) *before* `column_clusters()`, so the tiny nudge dominates the median diff and collapses the cluster threshold toward zero; genuine evenly-spaced casts each split into singleton columns (verified: `[0]*20+[1..5]` тЖТ 6 clusters instead of 1). Triggers whenever re-occupied stations (ties) coexist with normal spacing on a lat/lon/distance axis. **Fix:** compute clusters on the original pre-nudge x (median of strictly-positive diffs), nudge only for per-cluster edge geometry.

- **`perturb/netcdf_schema.py:192` тАФ CF/NetCDF.** `rho` carries `standard_name='sea_water_density'` but holds in-situ density **minus 1000** (`seawater.py:57`); its own `long_name` documents the offset. CF `sea_water_density` is the full ~1025 value, so a standard-name-driven consumer reads values ~1000 kg/m┬│ too small. (`sigma0` is correctly labeled `sea_water_sigma_theta`.) **Fix:** store full in-situ density (drop тИТ1000), or keep the anomaly and remove the CF `standard_name` (no CF name exists for in-situ-minus-1000).

- **`perturb/qc_rules.py:270` тАФ numerical.** In the pitch/W consistency rule, `W = smooth_fall_rate(P)` runs through `filtfilt`, which smears a single non-finite P sample across the **entire** record; `flag |= ~isfinite(W)` then marks the whole slow record flight-inconsistent (verified 60/60). Asymmetric with the range rule (one NaN тЖТ one sample). Reachable via hotel-injected P containing NaN. **Fix:** derive W-finiteness from the raw gradient of P (local), or use NaN-aware smoothing; drop `~isfinite(W)` from the smeared array.

- **`rsi/adapter.py:93-94,123` тАФ numerical.** The fixed-scalar `--speed` branch sets `pspd_rel = full(len, abs(speed))` with **no floor**, then divides shear by `pspd_rel**2`. `--speed 0` тЖТ inf/NaN shear baked into L1; tiny speed тЖТ huge over-amplification. The directly-analogous `"constant"` method (`speed.py:102`) *does* floor to `speed_cutout`. `--speed 0 --w-min 0` passes the section gate and detrend silently zeros non-finite samples. **Fix:** floor with the shared 0.05 m/s cutout (`max(abs(speed), SPEED_CUTOUT)`) and/or validate `--speed>0`.

- **`tests/test_window.py:201` тАФ test-gap.** `test_method2_iterative` asserts only `chi` finite/>0; never checks `kB`/`K_max_ratio`/range. This hides the `window.py:399` ChiFitResult field-order swap (Method-2 `kB` receives the `epsilon` value). **Fix:** assert Method-2 `kB` is finite and in a sane cpm range; cross-check against `_iterative_fit`/`l4_chi` with a synthetic known-kB Batchelor spectrum. *(Paired with the Minor `window.py:399` swap below.)*

- **`tests/test_perturb_combo.py:33` (+ rho value range) тАФ test-gap.** No test exercises differing `bin` grids (silent outer-join NaN-pad) or overlapping time grids (surviving duplicate timestamps); these gaps let the `combo.py` join/dedup defects ship. **Fix:** add a depth-combo test with mismatched bin vectors (assert error or correct regrid) and a time-combo test with overlapping ranges (assert strictly monotonic time).

## 3. Coverage & Caveats

**Audited.** RSI binary I/O and conversion (`p_file`, `channels`, `convert`, `deconvolve`, `profile`, `combine`), chi/epsilon physics (`chi`, `batchelor`, `fp07`, `l3_chi`, `l4_chi`, `window`, `scor160/l4`, `goodman`, `spectral`, `despike`, `nasmyth`), the perturb campaign pipeline (`pipeline`, `merge`, `trim`, `gps`, `hotel`, `ctd`, `binning`, `combo`, `seawater`, `qc_gate`, `qc_rules`, `netcdf_schema`, `config_base`), processing helpers (`ct_align`, `bottom`, `top_trim`, `mixing`, `epsilon_combine`, `chi_combine`), and the plotting subsystem (`profiles`, `eps_chi`, `scalar`, `layout`, `diagnostics`, `xaxis`, `mixing_look`, `viewer_base`, `quick_look`). Categories swept: correctness, numerical hazards (div-by-zero/NaN/filtfilt/dtype), oceanographic sign/unit/dateline errors, CF/NetCDF encoding, resource leaks, error-hiding, bin-edge/off-by-one handling, degenerate inputs, and test gaps.

**Confidence.** All findings were verified against the actual source (most reproduced empirically); high confidence overall, with a small number of explicitly medium-confidence items (`fp07_cal.py:58`, `channels.py:158` accel-half, `combo.py:96` dedup trigger frequency, `pipeline.py:1192` interrupted-write behavior, `gps.py` dateline real-world frequency, `chi/chi.py:579-605`). Note the report consciously **disconfirmed** several hypotheses: the epsilon CF standard_name *is* a real CF v89 entry (not a defect), the `top_trim` `len(bin_edges)<2` guard is *not* dead, the despike `fraction` is dead downstream, and `convert_accel`'s `adc_bits=0` default matches ODAS.

**Warrants direct human attention.**
1. **`files.merge=true` semantics** тАФ the two `merge.py` Majors mean enabling merge on this campaign would fuse 29 independent casts; confirm intended split-file detection before any user relies on merge.
2. **Probe-rejection low bias** (`epsilon_combine`/`chi_combine`) тАФ a systematic quantitative bias in the *core* epsilon/chi aggregate; a scientist should confirm the intended Lueck symmetric-rejection semantics and re-validate any published `epsilonMean`.
3. **numpy floor** (`pyproject.toml`) тАФ a one-line packaging decision (raise floor or shim) that gates whether the whole chi/epsilon subsystem runs on a conformant install.
4. **NaN-mean viscosity family** (`l3_chi.py:149/154`) and **median LnSigma** (`binning.py`) тАФ silent, scientifically meaningful corruption of chi and uncertainty under realistic inputs/config; prioritize the `nanmean` fixes.
5. **Geographic portability** тАФ the dateline (`gps`/`ctd`) and `bin_size` ULP-split (`combine.py`) defects are dormant for SN479 but will bite the first west-Pacific or non-binary-`bin_size` deployment; flag before reuse.
6. The audit reviewed code as-read on this branch; it did not run the full pipeline end-to-end on live data, so the few reachability judgments tied to real data distributions (dateline frequency, how often diss/chi depth ranges diverge) merit a confirming run.
